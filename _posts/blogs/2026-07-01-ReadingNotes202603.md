### Reading List in Past Two Months  

Here are the great articles I came across in the past two months.

#### Data Science & Analytics
1. [**Meta: A Blueprint for Valuing Content When A/B Tests Are Not an Option**](https://medium.com/@AnalyticsAtMeta/meta-a-blueprint-for-valuing-content-when-a-b-tests-are-not-an-option-7880bac721f1): How the Meta team uses DoubleML to measure the incremental impact of interacting with certain content.  
2. [**The Death of Dashboards**](https://medium.com/data-science-collective/the-death-of-dashboards-558ba0893926): The structural problem today is that dashboards answer the questions analysts anticipated, not always the questions users actually need to ask at the moment. And how LLM has transformed the workflow.  
3. [**Storytelling is the wrong skill for Dashboards — what to learn instead?**](https://martynas-mjdc.medium.com/storytelling-is-the-wrong-skill-for-dashboards-what-to-learn-instead-e231f151078d): why cannot we have storytelling in a dashboard, and what is that often overlooked critical skill specifically benefiting dashboard creators?  
4. [**How to Use a Time Series Foundation Model for Anomaly Detection**](https://ai.gopubby.com/how-to-use-a-time-series-foundation-model-for-anomaly-detection-718ba7090614): Pretrain a large time-series foundation model (TSFM) and apply it across many time series, out of the box for forecasting and then anomaly detection. Use Chronos-2 from AWS as an example.  
5. [**Data Bridge: How Netflix simplifies data movement**](https://netflixtechblog.medium.com/data-bridge-how-netflix-simplifies-data-movement-36d10d91c313): Data Bridge is Netflix’s unified control plane that simplifies and standardizes data movement across our diverse ecosystem of data stores.  


#### Data Career  
1. [**I’m a Data Analyst Who Barely Does Analytics Anymore.**](https://medium.com/data-science-collective/im-a-data-analyst-who-barely-does-analytics-anymore-895e16b7efe1): The traditional role that comprised writing SQL queries and building dashboards might no longer be in high demand. But hybrid roles will emerge.  
2. [**What Does Analytics Leadership Look Like in the Age of AI?**](https://eric-sandosham.medium.com/what-does-analytics-leadership-look-like-in-the-age-of-ai-3d8cf70ec768): Transform from manaaging insights to managing intelligence.  
3. [**10 Data + AI Predictions for 2026**](https://barrmoses.medium.com/10-data-ai-predictions-for-2026-adf2750cccba): Great observations of data industry trends related to AI.  


#### AI and LLM
1. [**A Complete Guide to Claude Code: Hooks**](https://ai.gopubby.com/a-complete-guide-to-claude-code-hooks-fcd3b262622d): The Hook mechanism lets you register processing logic that must run during Claude Code’s execution
2. [**Claude.md vs Skills vs Subagents vs Hooks: How to Choose the Right Claude Code Customization Layer**](https://medium.com/algomart/claude-md-vs-skills-vs-subagents-vs-hooks-how-to-choose-the-right-claude-code-customization-layer-903b75d061db): CLAUDE.md tells Claude how to behave all the time, Skills tell Claude how to think in specific situations, Subagents let Claude do work in a separate context, Hooks run automatically when events happen, and MCP gives Claude access to outside systems.  
3. [**Your AI Has Amnesia. Claude Subagents Are the Cure.**](https://medium.com/@nuno.roberto/your-ai-has-amnesia-claude-subagents-are-the-cure-7c46d46cb20a): The model’s attention degrades as the history grows, and how to use subagents to solve it. Detailed walkthrough of what is subagent, how to invoke them, subagent file structure, and common use patterns.  
4. [CLAUDE.md Best Practices](https://uxplanet.org/claude-md-best-practices-1ef4f861ce7c): 10 sections to Include in your CLAUDE.md.  
5. [**How I Continually Improve My Claude Code**](https://towardsdatascience.com/how-i-continually-improve-my-claude-code/): Tips to improve Claude Code efficiency, including daily cron job of a Claude Code conversation review skill, improving human interaction with coding agents, letting the agent ask you questions, etc.  
6. [**Self-Service Analytics, According to Anthropic**](https://medium.com/@annujackson/self-service-analytics-according-to-anthropic-722d8fbceab9): Analytics accuracy is a context and verification problem, not a code generation issue. How data foundations, sources of truth, skills, and validation help.  
7. [**5 Ways AI Is Reshaping Data Analytics (And what you can do about it)**](https://medium.com/@siglimumuni/5-ways-ai-is-reshaping-data-analytics-and-what-you-can-do-about-it-8897fe38a842): The role is shifting from data builders to data architects and storytellers.  
8. [**Why Traditional ETL is Dead: The Rise of Agentic Data Engineering**](https://medium.com/@seaflux/why-traditional-etl-is-dead-the-rise-of-agentic-data-engineering-34c4311f1d6c): How to build AI-ready data foundations without friction to replace traditional ETL.  
9. [**How Image Generation Actually Works**](https://pub.towardsai.net/how-image-generation-actually-works-d7638fca13d7): A high-level walkthrough of the two image generate methodology families: sculpting from noise, and writing an image like a sentence.  
10. [**Inside Meta’s Home Grown AI Analytics Agent**](https://medium.com/@AnalyticsAtMeta/inside-metas-home-grown-ai-analytics-agent-4ea6779acfb3): how to build a AI agent that performs routine data analysis tasks autonomously.
